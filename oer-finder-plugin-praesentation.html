Ausgangspunkt Ausgangspunkt & Förderziel
Ausgangspunkt & Förderziel

OER-Materialien sind fragmentiert

Das Problem

  • Inhalte verteilt auf isolierte Silos — Openverse, Wikimedia, ARASAAC usw.
  • Bequemlichkeit gewinnt: Gut nutzbare Angebote attraktiver als OER
  • Jede Bildungsanwendung muss aktuell jede Quelle selbst neu anbinden — enormer Aufwand

Die Idee & das Förderziel

  • Ein wiederverwendbares, offenes Plugin, welches möglichst viele OER-Quellen bündelt
  • Direkt integrierbar in bestehende Bildungsanwendungen
  • Aufbauend auf dem Nostr-Protokoll / Zusammenarbeit Edufeed

Architektur Architektur
Zwei Wege

Server-Proxy vs. Direct-Client

🖥

Server-Proxy

  • Voller Datenschutz — Dritte sehen keine Nutzer-IPs
  • Keine Zugriffssperre — Websites blockieren oft fremde Zugriffe (CORS), ein eigener Server umgeht das
  • Bildtransformation & Caching möglich
  • Eigener Server nötig; alle Nutzer teilen eine IP — Rate-Limits werden schneller erreicht
🌐

Direct-Client

  • Kein eigener Server nötig — sofort einsatzbereit
  • Schneller für Nutzende — kein Umweg über Server
  • Günstiger — keine Serverkosten
  • Zugriffssperren (CORS) & Datenschutz müssen bedacht werden

Einheitliches Datenformat

Der AMB-Standard

Alle Quellen liefern Daten in unterschiedlichen Formaten. Das OER Finder Plugin bildet jedes Ergebnis auf den AMB-Standard ab — so wird jede Quelle zum First-Class Citizen.

📐

Ein Standard

AMB definiert einheitliche Metadaten für alle Bildungsressourcen — unabhängig von der Herkunft.

🌐

Dezentral via Nostr

Der AMB Nostr Relay speichert OER-Metadaten als kryptographisch signierte Events — ohne zentrale Instanz.

🔄

Alles wird gemappt

Openverse, Wikimedia, ARASAAC — jede Quelle wird automatisch ins AMB-Format übersetzt.

Wirkung, Erfolge & Lernerfahrungen

Was wir gelernt haben

🎯

Wirkung bei Zielgruppen

  • Nutzertests über kits.app in Vorbereitung

Was gut funktioniert hat

Zentrale Lernerfahrungen

  • Architektur und Aufbau wiederverwendbarer JS-Komponenten mit Adaptersystem
  • Einfache OER-Integration ist in einer fragmentierten Landschaft eine Herausforderung aber möglich!
